## Project Overview

Bonsai is an HTML minifier written in pure Swift with zero external dependencies. It replicates the output of html-minifier-next's conservative preset. The public API is a single static method: `Bonsai.minifyHTML(_:) -> String`.

## Commands

```bash
just build     # swift build
just test      # swift test
just format    # swiftformat -swift-version 5 .
```

Run a single test:
```bash
swift test --filter BonsaiTests.BonsaiTests/testName
```

## Architecture

Two source files in `Sources/Bonsai/`:

- **Bonsai.swift** — Single-pass HTML minifier operating directly on a borrowed UTF-8 buffer (`UnsafeBufferPointer<UInt8>`). Inlines all tokenization and minification in one pass: whitespace collapsing, comment removal, boolean attribute collapsing, redundant attribute removal, doctype shortening. Uses depth counters (`preserveDepth`, `rawContentDepth`, `foreignContentDepth`) to track nesting context for `<pre>`/`<textarea>`, `<script>`/`<style>`, and `<svg>`/`<math>` respectively.

- **Constants.swift** — FNV-1a hash functions and `Set<UInt64>` hash-based lookup tables for void elements, boolean attributes, preserve-whitespace elements, raw content elements, default attribute values, and collapsible/removable attributes. Hash values are generated by `Scripts/generate_hashes.py`.

Other directories:

- `Tests/BonsaiTests/BonsaiTests.swift` — 183 test cases ported from html-minifier-next's test suite.
- `Benchmark/` — Simple benchmark harness using a fixture HTML file.
- `Scripts/generate_hashes.py` — Generates FNV-1a hash constants for Constants.swift.

## Code Style

- 2-space indentation, indent case statements (configured in `.swiftformat`)
- `Bonsai` is a caseless `public enum` used as a namespace; `minifyHTML` is its only `public static` method. All helpers are `private` free functions outside the enum
- Case-sensitive matching: `preserveWhitespaceHashes` uses exact-case FNV hashes (only lowercase `pre`, `textarea` match), while `rawContentElementHashes` and `voidElementHashes` use lowercased FNV hashes
- All parsing uses integer indexing into `UnsafeBufferPointer<UInt8>` — no `String.Index` except in the rare `findSubstringInString` helper for conditional comments
